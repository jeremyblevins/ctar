# Contributing to CTAR

Thank you for your interest in improving the Cyber Threat Assessment Report (CTAR) Framework.

## Guiding Principles

Contributions should support:

- Evidence-based analysis
- Analytical transparency
- Reproducibility
- Ethical cybersecurity
- Human verification of AI-assisted work

## How to Contribute

1. Open an issue describing the proposed change.
2. Discuss significant changes before implementation.
3. Submit a pull request with a clear description.
4. Ensure documentation follows the CTAR Style Guide.
5. Update the CHANGELOG when appropriate.

## Contribution Types

- Framework improvements
- Documentation
- Example CTARs
- Educational materials
- Bug fixes
- Editorial corrections

## What Not to Submit

- Live malware
- Exploit code
- Malicious tooling
- Copyrighted material without permission

By contributing, you agree that your work may be distributed under the project's CC BY 4.0 license.
